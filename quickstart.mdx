---
title: "Quickstart: build and run Claw Code in five minutes"
description: "Clone the repo, build the claw binary with cargo, verify your setup with claw doctor, and run your first prompt. Covers Unix and Windows PowerShell."
---

Claw Code is a build-from-source CLI. This guide walks you through cloning the repository, building the `claw` binary, and running your first prompt. The whole flow takes under five minutes on a machine with Rust already installed.

<Warning>
  **`cargo install claw-code` installs the wrong thing.** The `claw-code` crate on crates.io is a deprecated stub that places `claw-code-deprecated.exe` — not `claw`. Running it only prints `"claw-code has been renamed to agent-code"`. Do not use `cargo install claw-code`. Build from source using the steps below, or install the upstream binary with `cargo install agent-code`.
</Warning>

## Build and run

<Tabs>
  <Tab title="Linux / macOS">
    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/ultraworkers/claw-code
        cd claw-code/rust
        ```
      </Step>
      <Step title="Build the workspace">
        ```bash
        cargo build --workspace
        ```

        The `claw` binary is placed at `rust/target/debug/claw` after a successful debug build.
      </Step>
      <Step title="Set your API key">
        ```bash
        export ANTHROPIC_API_KEY="sk-ant-..."
        ```

        <Note>
          Claw requires an **API key** — a Claude subscription login is not a supported auth path. Get an API key at [console.anthropic.com](https://console.anthropic.com). If you prefer a different provider, see the [authentication guide](/configuration/authentication).
        </Note>
      </Step>
      <Step title="Run the doctor health check">
        ```bash
        ./target/debug/claw doctor
        ```

        `claw doctor` is the built-in preflight diagnostic. Run it every time after a fresh build or credential change to confirm that your API key is wired correctly, the binary resolves, and the runtime can reach the provider. Fix any issues it reports before running prompts.
      </Step>
      <Step title="Run your first prompt">
        ```bash
        ./target/debug/claw prompt "say hello"
        ```

        You should see the agent respond. From here you can open the interactive REPL by running `./target/debug/claw` without arguments.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Windows (PowerShell)">
    <Steps>
      <Step title="Install Rust">
        Download and run the installer from [rustup.rs](https://rustup.rs). Close and reopen your terminal when it finishes, then verify:

        ```powershell
        cargo --version
        ```
      </Step>
      <Step title="Clone the repository">
        ```powershell
        git clone https://github.com/ultraworkers/claw-code
        cd claw-code/rust
        ```
      </Step>
      <Step title="Build the workspace">
        ```powershell
        cargo build --workspace
        ```

        The binary is placed at `rust\target\debug\claw.exe` after a successful build.
      </Step>
      <Step title="Set your API key">
        ```powershell
        $env:ANTHROPIC_API_KEY = "sk-ant-..."
        ```

        <Note>
          Claw requires an **API key** — a Claude subscription login is not a supported auth path. Get an API key at [console.anthropic.com](https://console.anthropic.com).
        </Note>
      </Step>
      <Step title="Run the doctor health check">
        ```powershell
        .\target\debug\claw.exe doctor
        ```

        `claw doctor` is the built-in preflight diagnostic. Run it after every fresh build or credential change to confirm the binary and provider connection are healthy.
      </Step>
      <Step title="Run your first prompt">
        ```powershell
        .\target\debug\claw.exe prompt "say hello"
        ```

        Alternatively, use `cargo run` to skip the path lookup:

        ```powershell
        cargo run -- prompt "say hello"
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## JSON output for scripting

Pass `--output-format json` to get machine-readable output from any prompt. This is useful for automation, CI pipelines, and agent orchestration:

```bash
./target/debug/claw --output-format json prompt "status"
```

Example output:

```json
{
  "type": "result",
  "subtype": "success",
  "cost_usd": 0.0012,
  "is_error": false,
  "duration_ms": 843,
  "duration_api_ms": 801,
  "num_turns": 1,
  "result": "All systems operational.",
  "session_id": "session-20260410-001"
}
```
