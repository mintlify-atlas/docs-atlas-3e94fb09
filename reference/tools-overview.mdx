Claw exposes **40 tool specs** through `mvp_tool_specs()` in the `tools` crate. Every tool runs inside the current session and is gated by the active permission mode — `read-only`, `workspace-write`, or `danger-full-access`. Tools that exceed the current mode are blocked or escalated to the user before execution.

## Permission modes

| Mode | What it allows |
|------|---------------|
| `read-only` | Reads, searches, web fetches, LSP queries, task inspection |
| `workspace-write` | Everything above plus file writes, edits, config changes, notebooks |
| `danger-full-access` | Everything above plus bash, agents, task/team/cron creation, MCP auth |

Pass `--permission-mode` at startup or use `EnterPlanMode` / `ExitPlanMode` tools to switch modes inside a session.

### LSP tools

| Tool | Permission | Short description |
|------|-----------|-------------------|
| `LSP` | `read-only` | Query language server for symbols, references, diagnostics, definition, hover |

The `LSP` tool dispatches five `action` values: `symbols`, `references`, `diagnostics`, `definition`, and `hover`. Routing goes through `LspRegistry::dispatch()` in `runtime/src/lsp_client.rs`.

## Using `--allowedTools`

Restrict which tools are exposed to the model by passing a comma-separated list:

```bash
./target/debug/claw --allowedTools read_file,glob_search,grep_search prompt "find all TODOs"
```

Short aliases are supported: `read` → `read_file`, `write` → `write_file`, `edit` → `edit_file`, `glob` → `glob_search`, `grep` → `grep_search`.

## Tool count

`mvp_tool_specs()` currently exposes **40 tool specs**. The `Brief` alias is handled as an execution alias for `SendUserMessage` in `execute_tool()` but is not a separately listed spec. `TestingPermission` is test-only and not intended for production use.
