## Mock parity harness

The mock parity harness provides deterministic end-to-end coverage of CLI behavior without requiring live Anthropic credentials.

- **Crate:** `mock-anthropic-service` (`rust/crates/mock-anthropic-service/`)
- **Harness file:** `rust/crates/rusty-claude-cli/tests/mock_parity_harness.rs`
- **Scripted scenarios:** 10
- **Captured `/v1/messages` requests:** 19

### Harness scenarios

| Scenario | What it covers |
|----------|---------------|
| `streaming_text` | SSE streaming response decoding and display |
| `read_file_roundtrip` | Read-path execution and final synthesis |
| `grep_chunk_assembly` | Chunked grep tool output handling |
| `write_file_allowed` | File write success path |
| `write_file_denied` | Permission denial on file write |
| `multi_tool_turn_roundtrip` | Multi-tool assistant turns in a single conversation turn |
| `bash_stdout_roundtrip` | Bash execution and stdout capture |
| `bash_permission_prompt_approved` | Bash permission prompt — user approves |
| `bash_permission_prompt_denied` | Bash permission prompt — user denies |
| `plugin_tool_roundtrip` | Plugin-contributed tool execution path |

### Running the harness

```bash
cd rust

# Reproducible clean-environment run
./scripts/run_mock_parity_harness.sh

# Start mock service manually for ad hoc runs
cargo run -p mock-anthropic-service -- --bind 127.0.0.1:0

# Scenario checklist + PARITY mapping runner
python scripts/run_mock_parity_diff.py
```
